SVG 入门

很多时候我们都要使用矢量图形，或是在线描绘图形，这时H5的Canvas和SVG变成了SVG便成了不错的选择，关于两者的区别和优势这里不做讨论。这里只介绍SVG。 SVG是一种基于 XML 语法的图像格式，全称是可缩放矢量图（Scalable Vector Graphics）。可以理解为他是一段对图形的描述，所以它本质上是文本文件，体积小，有着不管放大多少都不会失帧的优点。

什么是SVG？

  • SVG 指可伸缩矢量图形 (Scalable Vector Graphics)
  • SVG 用来定义用于网络的基于矢量的图形
  • SVG 使用 XML 格式定义图形
  • SVG 图像在放大或改变尺寸的情况下其图形质量不会有所损失
  • SVG 是万维网联盟的标准
  • SVG 与诸如 DOM 和 XSL 之类的 W3C 标准是一个整体

SVG 的历史和优势
在 2003 年一月，SVG 1.1 被确立为 W3C 标准。
参与定义 SVG 的组织有：Sun公司（已被Oracle公司收购）、Adobe、苹果公司、IBM 以及柯达。

与其他图像格式相比，使用 SVG 的优势在于：

  • SVG 可被非常多的工具读取和修改（比如记事本）
  • SVG 与 JPEG 和 GIF 图像比起来，尺寸更小，且可压缩性更强。
  • SVG 是可伸缩的
  • SVG 图像可在任何的分辨率下被高质量地打印
  • SVG 可在图像质量不下降的情况下被放大
  • SVG 图像中的文本是可选的，同时也是可搜索的（很适合制作地图）
  • SVG 可以与 Java 技术一起运行
  • SVG 是开放的标准
  • SVG 文件是纯粹的 XML
  • SVG 的主要竞争者是 Flash。

与 Flash 相比，SVG 最大的优势是与其他标准（比如 XSL 和 DOM）相兼容。而 Flash 则是未开源的私有技术。

圆形 <circle>

<circle> 标签可用来创建一个圆，其余的css属性支持与rect差不多

  • cx和cy属性定义圆点的x和y坐标。如果省略cx和cy，圆的中心会被设置为(0, 0)
  • r属性定义圆的半径

椭圆 <ellipse>

<ellipse> 元素是用来创建一个椭圆，椭圆与圆很相似。不同之处在于椭圆有不同的x和y半径，而圆由一个r控制半径（毕竟是正圆形嘛）

  • CX属性定义的椭圆中心的x坐标
  • CY属性定义的椭圆中心的y坐标
  • RX属性定义的水平半径
  • RY属性定义的垂直半径

直线 <line>

<line> 用于画一条直线，先定义出开始点的横纵坐标，然后再定义出结束点的横纵坐标，两点之间一条直线（分明是线段）

  • x1 属性在 x 轴定义线条的开始
  • y1 属性在 y 轴定义线条的开始
  • x2 属性在 x 轴定义线条的结束
  • y2 属性在 y 轴定义线条的结束

曲线 <polyline>

这里的曲线其实就是将多个点互相连接实现一条曲折的线，而且可以将线段围起来形成图形并填充颜色

在线段之前也可以使用fill来填充颜色

文本 <text>

<text> 元素用于定义文本。它可以设置很多复杂的文本样式

其实大多数css样式 text都是支持的，我们可以用很多css的效果装饰文字，或者用一些html标签包裹文字以达成我们想要的效果

也可以使用transform属性来设置旋转


使用CSS中的writing-mode来设置其旋转


也可以使用<a>标签包裹，作为链接使用

Oracle

我们说的oracle即使oracle数据库，事实上oracle全称甲骨文股份有限公司(甲骨文软件系统有限公司)，是全球最大的企业级软件公司

oralce目前依然是占有率最大的关系型数据库，同样的还有sqlserver和mysql，但是其在分量和服务上都不如oracle，虽然说现在mysql有赶超oracle的趋势，但是现在oracle依然是关系型数据库的龙头。

Oracle不仅提供数据库服务,还提供例如数据支持、云计算、大数据等服务，连我们熟知的java，也属于oracle公司

关系型数据库

1970年 一个叫做codd的博士写了一篇叫做

的论文。在此文章里 创造性的提出了一个叫做关系形模型的概念
要点：

  1. 由对象和对象之间的关系组成的集合
  2. 对关系对应的一些操作
  3. 通过约束使数据的完整性保持精确和一致

简单来说，关系型数据库就是由一些二维表组成的（原文：A relational database is a collection of relations or two-dimensional tables）

表

事实上数据在存储中是以数据文件的方式存在的，但是我们使用时可以理解为数据是在各个表中的，一个关系型数据库由多个数据表组成（Table），数据表是关系数据库的基本存储结构，当我们使用sql（DQL）进行查询时数据依然以二维表的方式展现给我们

在我们遇到一些晦涩难懂的概念时，我们通常会将其模拟或抽象成一个比喻、一张图、或是一个故事，我们可以将这种方式称之为模型，而有一种叫做逻辑数据结构的模型，其中的实体关系模型（E-R）非常适合形容关系型数据库系统中表与表的关系

关系数据模型中，无论是是实体、还是实体之间的联系都是被映射成统一的关系—一张二维表，在关系模型中，操作的对象和结果都是一张二维表；
关系型数据库可用于表示实体之间的多对多的关系，只是此时要借助第三个关系—表，来实现多对多的关系；

举个例子：

一名职工可能对应多个产品，一个产品也可能对应多个职工，但是一个职工只能对应一个职工号、姓名、年龄，通过产品与职工、职工与职工号、产品与产品名等一对一、一对多甚至是多对多的关系，将数据之间互相关联，我们可粗略的理解为是关系型模型

为了减少数据库中的数据冗余，增强数据的易操作性，消除数据插入、删除异常等现象，关系模型要求关系必须是规范化的，这里不介绍三大范式，而是直接介绍一下满足范式的表与表之间存在的三种形式联系

一对一（1:1）的关联

一对一的关联从概念上表达的是一个数据对象与另外一个数据对象之间仅表现为一对一的关系；从逻辑上表达的是某个表中的一行数据行仅与另一个表中的某一数据行相对应。

一对多（1:N）的关联

一对多的关联从概念上表达的是一个数据对象与另外一个数据对象之间表现为一对多的关系；从逻辑上表达的是某个表中的一行数据行可与另外一个表中的多条数据行相对应。

多对多（M:N）的关联

多对多的关联从概念上表达的是一个数据对象与另外一个数据对象之间表现为多对多的关系；从逻辑上表达的是某个表A中的一行数据行可与另外一个表B中的多条数据行相对应，同时这个表A中的多条数据行又与表B中的某一行数据行相对应。

SQL

SQL – structured query language 结构查询语言

数据库对外的主要查询接口，是一个国际标准，几乎所有的数据库都支持，高效率，易学，是一种非过程化语言
非过程化语言- 只需要告诉数据库干什么就可以执行 如

我只需要输入当前sql语句 oracle server就会自动返回结果 不需要知道其过程，是一种非常高级的语言
Sql语言主要分为4类DML（包括DQL）、DDL、DCL

Dml：

数据操纵语言(其中select 可以称为数据查询语言dql 但是同一可称为 dml)

Insert 增加
update 更新
delete 删除

Ddl：

数据定义语言 主要用于创建一些对象

Create：创建

Alter：修改

Drop：删除

Rename：重命名

Truncate：清空

Dcl：

数据控制语言 主要定义一个用户的权限

Grant：授予

Revoke：撤销

其中我们最常用的就是DML语言，也就是我们常说的增删改查，也是之后着重要了解的东西
